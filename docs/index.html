Documentation

Everything you need to install, configure, and get the most out of metar‑cli. For a quick overview, see the home page.

Installation

The recommended install method is pipx, which gives metar its own isolated environment and makes it available globally without touching your system Python.

Recommended
$ pipx install metar-cli

Don't have pipx yet?

Install pipx
# macOS (MacPorts)
$ sudo port install pipx

# macOS (Homebrew)
$ brew install pipx

# Linux / other
$ pip install --user pipx

From source

For development or if you want to run the latest unreleased code:

Development install
$ git clone https://github.com/alexgc96/metar-cli.git
$ cd metar-cli
$ python3 -m venv .venv && source .venv/bin/activate
$ pip install -e .

Requires Python 3.10+. Dependencies (rich, requests, prompt_toolkit) are installed automatically.

Upgrading

$ pipx upgrade metar-cli

CLI Reference

Syntax
metar [--taf] [--raw] [-i] [--set-default ICAO] [ICAO ...]
OptionDescription
ICAO …One or more ICAO station codes (e.g. KJFK, MMML). Omit to use the default station.
--tafInclude the Terminal Aerodrome Forecast block below the METAR panel.
--rawPrint only the raw METAR string and exit. Useful for piping or logging.
-i / --interactiveLaunch the full-screen interactive dashboard.
--set-default ICAOSave a station as your default. Written to ~/.config/metar/config.
-h / --helpShow help and exit.

Examples

# Default station (from config or METAR_ICAO env var)
$ metar

# Single station
$ metar KJFK

# Multiple stations at once
$ metar KJFK KLGA KEWR

# With TAF forecast
$ metar KLAX --taf

# Raw METAR string only (pipeable)
$ metar MMML --raw

# Pre-flight: destination + alternate, with TAF
$ metar KBUR KLAX --taf

Configuration

Default station

Save a persistent home airport so metar (with no arguments) always shows your station:

$ metar --set-default KJFK

The station is written to ~/.config/metar/config as a plain text file.

Environment variable

The METAR_ICAO environment variable takes priority over the config file. Useful for scripting or per-session overrides:

# One-time override
$ METAR_ICAO=MMML metar

# Persistent in your shell profile
$ export METAR_ICAO=MMML

Priority order

  1. ICAO argument on the command line
  2. METAR_ICAO environment variable
  3. ~/.config/metar/config saved default
  4. Built-in fallback: MMML (Mexicali Intl)

Interactive Mode

Launch a full-screen weather dashboard that stays open and responds to keyboard commands. Built with prompt_toolkit.

$ metar -i
# or
$ metar --interactive

On launch, a station search box opens. Type any ICAO code and press Enter to load it. Once a station is displayed:

KeyAction
tToggle TAF forecast block
rShow / hide the raw METAR string
uRefresh — re-fetch live data for the current station
sOpen station search to load a different ICAO
cSave the current station as your default
qQuit

Understanding Output

Flight categories

The colored badge in the header indicates FAA standard flight rules based on ceiling and visibility:

VFR MVFR IFR LIFR
CategoryCeilingVisibility
VFR> 3,000 ft AGL> 5 SM
MVFR1,000 – 3,000 ft AGL3 – 5 SM
IFR500 – 999 ft AGL1 – 3 SM
LIFR< 500 ft AGL< 1 SM

The category is computed by AWC and returned directly in the API response — metar‑cli doesn't recompute it locally.

Wind rose

The ASCII compass rose shows wind direction. The arrows point toward the direction the wind is coming from. The center ✦ is the station. Wind speed and direction in degrees are shown below the rose.

Sparklines

The bottom row shows 6-hour trends for temperature, wind speed, and altimeter (QNH) as Unicode block charts. Data comes from the Iowa State University ASOS archive, sampled at roughly 12 points over the last 6 hours. Min/max range is labeled at each end.

Density altitude

DA is computed locally from OAT, altimeter setting, and station elevation using the standard ISA lapse rate formula. No external API call. Useful for performance planning — high DA means reduced aircraft performance even at low field elevations.

Remarks (RMK)

The remarks section is parsed from the raw METAR's RMK block:

CodeMeaning
SLPnnnSea-level pressure in hPa (e.g. SLP999 = 999.9 hPa)
TxxxxxxxxxPrecise temperature and dew point in tenths of °C
AO1 / AO2Station type — AO1 = no precipitation discriminator, AO2 = automated with discriminator
PK WND dddff/hhmmPeak wind during the hour — direction, speed, and time
PRESRR / PRESFRPressure rising/falling rapidly (>0.06 inHg in 3 hours)
$Maintenance check indicator — sensor or system needs attention

Weather phenomena

Present weather strings are decoded from ICAO intensity / descriptor / phenomenon codes:

CodeMeaning
-RA / RA / +RALight / moderate / heavy rain
TSRAThunderstorm with rain
SN / RASNSnow / rain and snow
BRMist (visibility 5/8 to 6 SM)
FGFog (visibility < 5/8 SM)
HZHaze
DZDrizzle
GR / GSHail / small hail
VCVicinity (within 10 SM of the field but not at it)

Safety reminder — metar‑cli decodes present weather for situational awareness. Always verify conditions via official sources before flight. A single surface METAR doesn't capture en-route conditions, SIGMETs, AIRMETs, or winds aloft.

Data Sources

No API key is required for any data source metar‑cli uses.

METAR & TAF — Aviation Weather Center

All primary weather data comes from the AWC public API operated by NOAA/NWS:

DataEndpoint
METAR (current)aviationweather.gov/api/data/metar?ids=MMML&format=json
TAF (forecast)aviationweather.gov/api/data/taf?ids=MMML&format=json

Key METAR fields used: temp, dewp, wdir, wspd, wgst, visib, altim, clouds, fltCat, wxString, rawOb, elev, obsTime.

Historical observations — Iowa State Mesonet

The 6-hour sparkline history uses the ISU Environmental Mesonet ASOS archive. Free, no auth:

https://mesonet.agron.iastate.edu/cgi-bin/request/asos.py
  ?station=MMML
  &data=tmpf,drct,sknt,alti
  &format=onlycomma
  &missing=null
  &tz=UTC

Fields used: tmpf (°F → °C), sknt (kt), alti (inHg). Up to 76 observations per 6h window are sampled to 12 points for the sparkline display.

Computed locally

ValueMethod
Density altitudeOAT + altimeter + station elevation via ISA lapse rate formula. No external call.
Weather phenomenaParsed from wxString using ICAO intensity/descriptor/phenomenon codes.
RemarksParsed from rawOb RMK section (SLP, T/Td, station type, peak wind, pressure tendency).

Roadmap

These are ideas and plans — not commitments. The near-term list is where active work happens.

Near-term
Crosswind calculator
Fetch runway headings from OurAirports, compute headwind and crosswind components from current METAR wind. CLI flag --xwind RWY28 and interactive key x.
SIGMETs / AIRMETs
Active significant meteorological information for a station's region. Same AWC API, no auth required.
PIREPs
Pilot reports within a configurable radius of the station. "What are pilots actually seeing" layer.
Winds aloft
Temperature and wind at pressure altitudes (3k–30k ft). Natural companion to density altitude.
Medium-term
Multi-station comparison table
Side-by-side view for metar KJFK KLGA KEWR — flight categories, wind, visibility in a compact table.
ATIS text decode
Parse raw ATIS text into a readable summary.
NOTAMs
FAA/ICAO NOTAM API integration. Lower priority due to high noise ratio in raw NOTAM data.
Longer-term
Flight planning session mode
Enter a route (e.g. MMML KTUS KPHX), pull METAR + TAF for each waypoint, show the en-route weather chain in one view.
Config profiles
Multiple default stations (home base, work airport, cross-country). Toggle with a key in interactive mode.
Offline cache persistence
Write the last fetch to ~/.cache/metar/ on exit, reload on next start. Useful on spotty connections.
